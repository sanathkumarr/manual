After reboot:

ls -l                     # Verify files exist

üåç Part 2: Copy EBS Snapshot to Another Region (Cross-Region Transfer)

1Ô∏è‚É£ Create Snapshot from Data Volume
- Go to Volumes ‚Üí Actions ‚Üí Create Snapshot
- Add name/tag: e.g., data-snapshot-nvirginia

2Ô∏è‚É£ Copy Snapshot to Another Region
- Go to Snapshots ‚Üí Select Snapshot ‚Üí Actions ‚Üí Copy

3Ô∏è‚É£ Switch to Ohio Region and Create Volume from Snapshot
- Switch to Ohio (us-east-2) region.
- Go to Snapshots, verify the copied snapshot exists.
- Click Actions ‚Üí Create Volume:
  - From snapshot: data-snapshot-ohio
  - AZ: e.g., us-east-2a

4Ô∏è‚É£ Launch a New EC2 Instance in Ohio

- Choose same AZ as volume (e.g., us-east-2a).

5Ô∏è‚É£ Attach Volume to Ohio Instance
- Go to Volumes ‚Üí Select Volume ‚Üí Attach Volume
  - Choose instance.
  - Device name: /dev/xvdb

6Ô∏è‚É£ SSH into Ohio EC2 and Mount Volume
